# IMPLEMENT — llm-gateway (паста для LLM-ассистента)

Скопируйте в чат с LLM-ассистентом, заполнив <…>.

---

Ты настраиваешь **llm-gateway** — единую OpenAI-совместимую точку к моделям на
базе LiteLLM. Конфиг-образец — `config.example.yaml`.

## Мои провайдеры
- Основной: <OpenAI | Anthropic | Yandex GPT | self-hosted vLLM | OpenRouter>
- Модель: <gpt-4o | claude-… | …>
- Фолбэк: <провайдер/модель или нет>
- Виртуальные ключи на сервис нужны: <да/нет> (если да — нужен postgres)

## Сделай
1. `config.yaml` из образца: model_list с моими провайдерами, fallback,
   master_key и api_key — только через env.
2. Список env-переменных, которые мне завести.
3. Подключение к платформе: в `.env` платформы выставить
   `LLM_BASE_URL=http://llm-gateway:4000/v1` и `LLM_API_KEY=<virtual key>`.
4. Покажи команду генерации виртуального ключа (`gen_key.sh`).

Инвариант: ни один ключ провайдера не хардкодится; агент знает только адрес
шлюза и виртуальный ключ, не реальные ключи провайдеров.
