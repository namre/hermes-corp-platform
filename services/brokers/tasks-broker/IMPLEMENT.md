Ты реализуешь **tasks-broker** для корпоративной агентской платформы. Сейчас это
заглушка (`main.py`). Замени на реальную интеграцию, СОХРАНИВ контракт.

## Моя система
- Таск-трекер: <Kaiten | Jira | YouTrack | Asana | другое>
- Базовый URL API: <https://mycorp.kaiten.ru/api/latest>
- Аутентификация пользователя в трекере: <персональный API-токен | OAuth>

## Контракт (не менять)
- `/health`, `/mcp` (диспетчер инструментов из `TOOLS.md`).
- Доступ только из внутренней сети, проверка `BROKER_INTERNAL_AUTH`.
- Личность — email в заголовке от proxy.

## Обязательные инварианты
1. Доступ к задачам — **от имени пользователя** (его токен), не общим ключом.
2. Токены — зашифрованы at-rest мастер-ключом из env.
3. Пишущие операции (create/comment/move) — через **two-step commit**:
   готовим изменение → `confirm_token` → отдельный `commit` применяет.
4. `TASKS_BASE_URL` и все секреты — только через env.

## Что реализовать
- connect/status/disconnect (привязка персонального токена).
- Чтение: my_tasks (с обогащением срочности по дедлайну), find_card, get_card,
  list_spaces, list_boards.
- Запись через commit: create_card, add_comment, move_card.

Сначала план и список env-переменных, потом код. Сохрани формат полей задач из
заглушки (`id`, `title`, `assignee`, `status`).
