# tasks-broker — каталог инструментов (образец: Kaiten)

Заглушка отдаёт фейковые задачи из памяти. Боевой образец на базе Kaiten даёт
агенту такой набор MCP-инструментов. Чек-лист для реализации под Jira/YouTrack/Asana.

| Инструмент | Назначение | Тип |
|---|---|---|
| status | подключён ли пользователь к трекеру | read |
| connect / disconnect | привязать/отвязать персональный API-токен | write |
| my_tasks | мои карточки с обогащением (срочность по дедлайну) | read |
| find_card | поиск карточек по запросу | read |
| get_card | детали карточки | read |
| list_spaces / list_boards | навигация по пространствам/доскам | read |
| create_card | создать карточку | write (через commit) |
| add_comment | комментарий к карточке | write (через commit) |
| move_card | перенести по колонкам | write (через commit) |
| commit | подтверждение мутаций по confirm-token | write-gate |

## Принцип подтверждения мутаций
Пишущие операции не применяются сразу: агент готовит изменение и получает
`confirm_token`, реальное применение — отдельным `commit`. Это «человек жмёт
подтвердить», а не «агент молча меняет трекер».

## Что прописать в своей конфигурации
- `TASKS_BASE_URL` = адрес API трекера (напр. `https://<домен>.kaiten.ru/api/latest`).
- Персональный API-токен пользователя (вводится через `connect`, хранится зашифрованным).
- `TASKS_TOKEN_MASTER_KEY` = `openssl rand -hex 32`.
