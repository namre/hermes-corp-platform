# IMPLEMENT — corp-dir-broker (паста для LLM-ассистента)

Скопируйте в чат с LLM-ассистентом, заполнив <…>.

---

Ты реализуешь **corp-dir-broker** — каталог сотрудников. Сейчас читает локальный
`sample_directory.json`. Замени `load_directory()` на реальный источник, СОХРАНИВ
эндпоинты и формат полей.

## Мой источник правды о сотрудниках
- Система: <HR-портал | LDAP | Active Directory | Google Directory | API>
- Доступ: <URL + сервисный токен в env>
- Где взять маппинг Telegram ID ↔ сотрудник: <…>

## Контракт (не менять)
- `/health`, `/users/by-email`, `/users/by-telegram`, `/users/search`, `/org-structure`.
- Поля: `email`, `full_name`, `position`, `org_unit`, `telegram_id`, `manager_email`.
- Проверка `BROKER_INTERNAL_AUTH`, доступ только из внутренней сети.

## Инварианты
1. **Кэш с периодическим обновлением** — не дёргать HR-API на каждый запрос
   (в проде обновлялось несколько раз в день). Индексы по email/telegram/username.
2. `normalize()` приводит ответ вашей системы к полям контракта.
3. Секреты — только env.

Сначала план и env-переменные, потом код. Это базовый блок — от него зависит
резолв личности во всей платформе, поэтому делаем первым.
