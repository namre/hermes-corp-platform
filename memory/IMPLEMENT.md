# IMPLEMENT — память (паста для LLM-ассистента)

Скопируйте в чат с LLM-ассистентом, заполнив <…>.

---

Ты подключаешь **память** к корпоративной агентской платформе по схеме
«личный банк + корпоративный read-only банк».

## Мой провайдер памяти
- Сервис: <Hindsight | свой векторный сервис | другое>
- Адрес (внутренняя сеть): <http://memory:PORT>
- База знаний для корп-банка: <Wiki | Confluence | Notion | …>

## Сделай
1. Адаптер к контракту: `POST /retain`, `POST /recall` (см. README).
2. Личный банк `user:<id>` с auto-retain/auto-recall, изоляция по пользователю.
3. Корп-банк `corp:<org>` read-only; `retain` в него отклоняется на уровне сервиса.
4. Конвейер наполнения корп-банка из базы знаний: страница → факты с
   метаданными (документ, ссылка, дата, версия). Опиши, как запускать (cron).
5. Список env: `MEMORY_PROVIDER_URL`, `MEMORY_PROVIDER_TOKEN`, `MEMORY_CORP_BANK`.

## Инварианты
- Личные банки изолированы; чужой прочитать нельзя.
- Корп-банк только для чтения; ответы агента — со ссылкой на источник.
- Транспорт до пров() — приватный, не публичный интернет.

Сначала план и env-переменные, потом код адаптера.
