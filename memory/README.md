# Память — личная + корпоративная (образец: Hindsight)

Память вынесена во внешний сервис, к которому агент обращается за фактами. В
шаблоне выключена по умолчанию (`MEMORY_PROVIDER_URL` пуст). Боевой образец —
связка с **Hindsight**: личный банк сотрудника + корпоративный банк компании.

## Два контура

### Личная память — `user:<id>` (read/write)
- **auto-retain:** агент в фоне извлекает факты из реплик и кладёт в банк
  пользователя. Команда «запомни» не нужна.
- **auto-recall:** в новой сессии агент сам подтягивает релевантное к вопросу.
- Изолирован на пользователя: чужой банк недоступен, в т.ч. админу. Привязка по
  `runtime_id`/email; каждый user-runtime видит только свой банк.

### Корпоративная память — `corp:<org>` (read-only)
- Наполняется **отдельным конвейером** из базы знаний компании (Wiki/Confluence/
  Notion): страница → факты с метаданными (документ, ссылка, дата правки, версия).
- Агент обязан отвечать **со ссылкой на источник**, а не пересказом по памяти.
- Запись из агента физически невозможна: read-only форсится и на стороне
  провайдера, и на стороне агента (инструмент записи в этот банк агенту не показан).

## Контракт провайдера (минимум)

```
POST /retain   {bank, text, entities?}     -> {ok}      # только личные банки
POST /recall   {bank, query, top_k}        -> {facts[]} # личный или корп
```
Корпоративный банк обязан отклонять `retain` на уровне сервиса.

## Что прописать в конфигурации (без секретов)
- `MEMORY_PROVIDER_URL` — адрес сервиса памяти (внутренняя сеть).
- `MEMORY_PROVIDER_TOKEN` — сервисный токен (env).
- `MEMORY_CORP_BANK` — имя корп-банка (напр. `corp:mycorp`).
- Политика именования банков: `user:<runtime_id>`, `project:<slug>`, `corp:<org>`.
- Транспорт до внешнего провайдера — приватный (VPN/mesh), не публичный интернет.

## Naming policy (из боевого опыта)
Единые имена банков спасают от путаницы при росте: `user:*` — личные,
`project:*` — проектные, `corp:*` — корпоративные read-only. Реестр банков
полезно вести явно.

Инструкция для LLM-ассистента собрать связку — [IMPLEMENT.md](IMPLEMENT.md).

## Зачем строго с источниками
Главный риск ИИ — отвечать уверенно и неправильно. Лекарство — заставить агента
не «вспоминать», а **цитировать**: документ, дата, версия. Обновили регламент
вчера — агент завтра отвечает по новому, со ссылкой.
